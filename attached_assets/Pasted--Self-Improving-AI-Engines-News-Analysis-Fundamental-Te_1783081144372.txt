# Self-Improving AI Engines: News Analysis, Fundamental+Technical Fusion & Genome Evolution

## Goal
Transform all AI engines (Signal Engine/HERMES, Swing Watch, IPO Radar, Scalp Radar/APEX) into fully **self-learning AND self-improving** systems that:
1. **Signal Engine (HERMES)**: Integrate deep news analysis + VCP pattern + best fundamental + technical signals → composite scoring
2. **All Engines**: Self-improving genome evolution that mutates scan parameters until achieving **5-10% avg returns**
3. **Swing Watch / IPO Radar / Scalp Radar**: Currently lack self-improving capabilities — add genome evolution + outcome tracking + learning cycles

## Open Questions

> [!IMPORTANT]
> 1. **Database migration**: Adding new tables (`swing_outcomes`, `ipo_outcomes`, `news_impact_log`) requires a database migration. Should I run `db:push` automatically after creating the schema changes?

> [!IMPORTANT]  
> 2. **News API rate limits**: Enhanced news analysis uses Gemini API for deep sentiment. The free Gemini tier has rate limits. Should we use `gemini-flash-latest` (faster, cheaper) or `gemini-pro` (more accurate) for news analysis?

> [!WARNING]
> 3. **Initial cold start**: All genome evolution requires at least 10 completed trade outcomes to begin self-improving. Until the system accumulates enough data, it will use default parameters. This is by design — it prevents premature optimization on insufficient data.
