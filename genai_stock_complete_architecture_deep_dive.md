# GenAI-Stock (Wealthy Whales) — Complete System Architecture & Deep Dive

GenAI-Stock is a professional-grade quantitative stock intelligence and scanning platform focused on the Indian equities markets (NSE and BSE). It leverages real-time data scraping, Technical and Fundamental analytics, and dual self-learning background AI engines (HERMES and FUGU) to generate high-conviction trade recommendations, score assets, and evolve scoring weights based on historical market outcomes.

## 2. Database Schema & Data Models

All database tables are defined using Drizzle ORM in [shared/schema.ts](file:///c:/Users/USER/Desktop/genai-stcok/shared/schema.ts). The database supports user authentication, community feeds, and extensive state tracking for both self-learning AI engines.

### 2.1 Core Application Tables
- **`users`**: Manages user profiles, admin flags, and Telegram integration hooks (`telegramChatId`).
- **`sessions`**: Storage for active user login sessions (PostgreSQL-backed for Replit Auth).
- **`stockRecommendations`**: Holds recommendations manually published by administrators (with Targets, Stop-loss, and rationales).
- **`chatMessages`**: Tracks session-grouped messaging history for the AskAI chatbot interface.
- **`scannerData`**: Caches and indexes the results of standard scanners (Swing and IPO).
- **`newsItems`**: Stores scraped news articles with sentiments.
- **`marketDataCache`**: General cache for rate-limited financial APIs.

### 2.2 HERMES AI Engine Tables
- **`hermesSnapshots`**: Daily snapshot logs containing technical parameters (RSI, SMAs, MACD, ADX, ATR), fundamentals (PE, ROE, ROCE, D/E, OPM, PEG, Market Cap, Div Yield), and the generated HERMES Score/Verdict.
- **`hermesOutcomes`**: Triggers forward price tracking (5d, 10d, 20d) for each snapshot. Once the time elapsed matches the criteria, forward returns are computed, and the trade is classified as a `WIN`, `LOSS`, or `NEUTRAL`.
- **`hermesWeights`**: Versioned table keeping track of active weights. Every Sunday learning cycle deactivates the current version, creates a new row with adjusted weight matrices, and stores the tested historical accuracy.
- **`hermesRegimeLog`**: Historical log tracking Nifty-based market classification trends (`TRENDING_UP`, `TRENDING_DOWN`, `RANGING`, `VOLATILE`) and overall market breadth.

### 2.3 FUGU AI Engine Tables
- **`fuguSnapshots`**: Logs multi-agent LangGraph outputs for each potential candidate. Stores feature sets, sub-agent scores (Technical, Pattern, Candlestick, Fundamental, Sector, Macro), similarity metrics to past winners/losers, final Fugu Scores, and elite rationale justifications.
- **`fuguOutcomes`**: Similar to Hermes outcomes but checks over a longer, multi-frequency scope: **5d, 10d, 20d, 30d, 60d, and 90d forward return horizons**, including drawdown stats (`maxDrawdown`) and volatility metrics.
- **`fuguFactorWeights`**: Evolving factor weights config vector (versioned).
- **`fuguPatternStats` / `fuguCandlestickStats`**: Cumulative records tracking the win-rate statistics of individual chart patterns (e.g., Cup & Handle) and candlestick structures (e.g., Bullish Engulfing) over time.
- **`fuguSectorStats`**: Sector success statistics tracking win rates and momentum scores dynamically.
- **`fuguLearningMemory`**: Self-improvement findings and rules generated by Gemini after analyzing winning and losing trades.
- **`fuguElitePicks`**: Top 5/10/20 ranked stocks selected daily.

## 5. Machine Learning Engines (Deep Dive)

GenAI-Stock's key differentiator is its dual machine learning stack. Both engines run concurrently via scheduled cron-like loops.

### 5.1 HERMES AI Engine

HERMES evaluates a broad list of stocks against a set of 20+ feature rules. It adjusts its internal weights dynamically based on whether its recommendations result in profitable returns.

```
       [ Daily Scan ]
              |
              v
[ Evaluate 20+ Feature Rules ]  <--  [ Loaded Weights (hermesWeights) ]
              |
              v
     [ Calculate Score ]
              |
              v
   [ Insert Snapshot & Outcome ]
              |
              +----------------------+
              | (After 5/10/20 Days) |
              v                      v
      [ Outcome Tracker ] ---> [ Evolve Weights (Weekly) ]
```

#### A. Daily Scanner ([hermesEngine.ts:L297-L493](file:///c:/Users/USER/Desktop/genai-stcok/server/hermesEngine.ts#L297-L493))
Runs weekdays after market close. For each stock in [nseUniverse.ts](file:///c:/Users/USER/Desktop/genai-stcok/server/nseUniverse.ts):
1. Fetches historical candles and fundamentals.
2. Extracts indicators (RSI, SMA20/50 crossovers, MACD, ADX, ATR, RVOL).
3. Compiles a feature state.
4. Calculates the weighted score based on active weights.
5. Saves a row to `hermesSnapshots` and initiates a tracking entry in `hermesOutcomes`.

#### B. Outcome Tracker ([hermesEngine.ts:L499-L609](file:///c:/Users/USER/Desktop/genai-stcok/server/hermesEngine.ts#L499-L609))
Checks for pending outcome rows:
- **5-Day return**: If date > 7 calendar days ago, updates the quote price. If return $\ge 3\%$, outcome is a `WIN`. If return $\le -3\%$, it's a `LOSS`.
- **10-Day and 20-Day returns**: Checks at 14 and 28 calendar days.

#### C. Evolving weights ([hermesEngine.ts:L615-L750](file:///c:/Users/USER/Desktop/genai-stcok/server/hermesEngine.ts#L615-L750))
Runs weekly on Sunday:
1. Gathers completed outcomes (where at least 5-day return is filled).
2. For each rule condition (e.g. `rsi14_zone`, `low_debt`), calculates the win rate when that condition was true.
3. Computes the target weight for each feature:
   $$\text{targetWeight} = \text{winRate} \times \text{defaultWeight} \times 2$$
4. Blends the target weight with the old weight using exponential smoothing (70% old, 30% new):
   $$\text{blendedWeight} = (0.7 \times \text{oldWeight}) + (0.3 \times \text{targetWeight})$$
5. Clamps individual weights to $[0.01, 0.20]$, normalizes them to sum to $1.0$, increments the version, and saves the configuration.

#### D. Regime Classification ([hermesEngine.ts:L756-L800](file:///c:/Users/USER/Desktop/genai-stcok/server/hermesEngine.ts#L756-L800))
Uses Nifty 1-week and 1-month returns to log the active market phase (`TRENDING_UP`, `TRENDING_DOWN`, `VOLATILE`, or `RANGING`), providing context for weight adjustments.

### 5.2 FUGU AI Engine

FUGU is a multi-agent self-learning system that utilizes a **LangGraph state machine** to rate stocks. It incorporates cosine similarity comparisons against historical winners, AI-extracted news catalysts, and sector momentum.

```
                  [ Gather Candidates ]
                 (Swing, IPO, Patterns, Screener)
                            |
                            v
              [ LangGraph Agent Pipeline ]
                            |
                            v
       +--------------------+--------------------+
       |                                         |
       v (Technical Agent)                       v (Sector Agent)
  Evaluate RSI, RVOL,                        Lookup historical
  BB Bandwidth Squeezes                      sector win rate & momentum
       |                                         |
       v (Pattern Agent)                         v (Macro Agent)
  Recognize Chart Patterns                   Calculate Nifty return
  (Double Bottom, Cup & Handle)              direction
       |                                         |
       v (Candlestick Agent)                     v (Sentiment Agent)
  Identify Hammer, Engulfing                 Gemini extracts news
  candle formations                          catalyst & sentiment
       |                                         |
       +--------------------+--------------------+
                            |
                            v
                 [ Similarity Agent ]
                 Cosine similarity to past
                 WIN/LOSS feature vectors
                            |
                            v
                 [ Elite Ranker Agent ]
                 Blends scores with active weights.
                 Gemini writes elite rationale.
                            |
                            v
                 [ Elite picks & Outcomes ]
```

#### A. Candidate Gathering ([fuguEngine.ts:L697-L775](file:///c:/Users/USER/Desktop/genai-stcok/server/fuguEngine.ts#L697-L775))
FUGU filters candidates from multiple input screeners to focus on the top 40 high-probability stocks:
1. **Swing Scanner**: Technical parameters filter mid/small caps.
2. **IPO Base Scanner**: Caches stocks listed within 110 days forming tight consolidations.
3. **Pattern Scanner**: Technical patterns (Cup & Handle, Flags).
4. **Smart Screener**: NLP queries.

#### B. The LangGraph Pipeline ([fuguEngine.ts:L112-L145](file:///c:/Users/USER/Desktop/genai-stcok/server/fuguEngine.ts#L112-L145))
Each candidate stock is processed sequentially through the following nodes:
- **`technical`**: Rates standard technical indicators. Evaluates volatility squeezes (BB bandwidth proxy: $2 \times \text{ATR}_{14} / \text{SMA}_{20}$).
- **`pattern`**: Identifies chart setups. Integrates with historical pattern performance data.
- **`candlestick`**: Detects candlestick reversals (Morning Star, Engulfing, Hammer, Inside Bar) and checks volume confirmation.
- **`fundamental`**: Performs fundamental analysis (DuPont check, high ROE, margins, low debt, reasonable PE).
- **`sector`**: Pulls sector strength, blending historical win rates and current momentum.
- **`macro`**: Adjusts scoring based on Nifty's direction.
- **`sentiment`**: Parses recent news headlines and feeds them to Gemini. Gemini returns a sentiment score ($0-100$) and a concise catalyst summary (e.g., *"Strong Q4 earnings growth"*).
- **`similarity`**: Calculates cosine similarity between the candidate's feature vector and historical winning/losing trades. The feature vector includes normalized values for RSI, ROE, D/E, PE, and RVOL:
  $$\cos(\theta) = \frac{\mathbf{A} \cdot \mathbf{B}}{\|\mathbf{A}\| \|\mathbf{B}\|}$$
- **`eliteRanker`**: Computes the final Fugu Score by combining the sub-scores with the active weight vector. For high-scoring stocks ($\text{Score} > 72$), Gemini writes a concise institutional-grade rationale.

#### C. Evolving Weights ([fuguEngine.ts:L1268-L1391](file:///c:/Users/USER/Desktop/genai-stcok/server/fuguEngine.ts#L1268-L1391))
During the weekly learning cycle, FUGU optimizes its weights by calculating the Pearson correlation coefficient between each sub-agent's score and the actual 20-day returns:
$$r = \frac{\sum (x - \bar{x})(y - \bar{y})}{\sqrt{\sum (x - \bar{x})^2 \sum (y - \bar{y})^2}}$$
The weights are updated using exponential smoothing (80% old, 20% new) and normalized to sum to $1.0$:
$$\text{blendedWeight} = (0.8 \times \text{oldWeight}) + (0.2 \times (r \times 2))$$

#### D. Gemini Learning Memory ([fuguEngine.ts:L1192-L1266](file:///c:/Users/USER/Desktop/genai-stcok/server/fuguEngine.ts#L1192-L1266))
Gemini analyzes the feature profiles of the top 5 winning trades and top 5 losing trades. It generates structured insights outlining why winners succeeded and why losers failed, logging these findings to `fuguLearningMemory`.

## 6. How the Schedulers Start

When the application registers its routes via `registerRoutes(app)` in [routes.ts](file:///c:/Users/USER/Desktop/genai-stcok/server/routes.ts):
1. **`startHermesScheduler()`** is called ([routes.ts:L1966](file:///c:/Users/USER/Desktop/genai-stcok/server/routes.ts#L1966)), initiating the HERMES cron checks (e.g., checking every 30 minutes for the daily scan and running weekly learning cycles).
2. **`startFuguScheduler()`** is called ([routes.ts:L2102](file:///c:/Users/USER/Desktop/genai-stcok/server/routes.ts#L2102)), initiating the FUGU pipelines.
